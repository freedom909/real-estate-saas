## Key Fix: "show my bookings" routing (2026-06-23)
- **Problem**: "show my bookings" returned `(action: GENERAL)` with empty artifacts
- **Root cause**: `SemanticExtractor` uses the **listing** `LLMExtractor` as LLM fallback. The listing extractor's prompt didn't include `GET_MY_BOOKINGS`, so LLM returned `domain: "GENERAL"`. `resolveActionFromEntities` fixed the action to `GET_MY_BOOKINGS` but **domain stayed GENERAL** → router sent to `GeneralAgent` instead of `BookingAgent`.
- **Fix in `listing/llm.extractor.ts`**:
  1. Added `resolveDomainFromAction()` — corrects domain based on resolved action (booking actions → `BOOKING`)
  2. Added `GET_MY_BOOKINGS` to LLM prompt's available actions
  3. Added "show my bookings" / "list my bookings" examples to prompt
- **Note**: The `IntentGateway` keyword matcher and booking `llm.extractor.ts` both had correct overrides, but neither was in the active code path — `SemanticExtractor` only uses listing extractor as LLM fallback.
