# AI Platform Context — Wisdom Refactor

## Architecture
- **Entry**: `src/wisdom/index.ts` — Express + Apollo Server (subgraph) on port 4200
- **Dev script**: `npm run dev:ai` or `npm run dev:wisdom` (both point to `src/wisdom/index.ts`)
- **DI**: tsyringe container, registered via `src/wisdom/container/registrations/wisdom.register.ts`
- **Auth**: `getUserFromToken(req)` extracts JWT from `Authorization: Bearer <token>` header
- **Flow**: Resolver → AIRequestFactory → WisdomOrchestrator → SemanticExtractor → ReferenceResolver → AgentRouter → Agent → Memory Update → Response

## src/wisdom/ — The Intelligence Engine

### Pipeline
```
User Request
  → SemanticExtractor (Rule → MessageRule → LLM fallback)
  → ReferenceResolver (ordinal/pronoun resolution, auto-search)
  → AgentRouter (domain → agent mapping)
  → Agent.execute()
  → BookingStateUpdater (memory)
  → WisdomResponse
```

### DI Tokens
- `WISDOM_TOKENS.semanticExtractor` — composite extractor
- `WISDOM_TOKENS.extractors.{rule,messageRule,llm}Extractor`
- `WISDOM_TOKENS.referenceResolver`
- `WISDOM_TOKENS.router` — AgentRouter
- `WISDOM_TOKENS.agents.{listing,booking,general}Agent`
- `WISDOM_TOKENS.memory.bookingStateUpdater`
- `WISDOM_TOKENS.orchestrator` — WisdomOrchestrator
- `WISDOM_TOKENS.chatUseCase`

### External Dependencies (registered in src/wisdom/index.ts)
- `registerListingDependencies()` — registers ListingRepository, SearchListingUseCase, GetListingUseCase, etc.
- `BookingRegister()` — registers SequelizeBookingRepository, CreateBookingUseCase, CancelBookingUseCase, etc.
- `OpenAIAdapter` — registered under both `TOKENS_AI.OpenAIAdapter` and `TOKENS_AI.usecase.llmProvider`
- `registerAuditDependencies()`, `cacheContainer()` — audit + Redis cache
