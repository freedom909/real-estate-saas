# Authorization System Design — minshuku-saas

## Problem Summary

- Gateway extracts JWT but **never rejects** unauthenticated requests (always `next()`)
- 11 of 16 subgraphs have **zero auth protection**
- 5 incompatible role enums across backend/frontend/GraphQL
- Policy engine (`src/rbac/policy.engine.ts`) exists but is **never called**
- No resource-level authorization (booking/payment/review access by ID without ownership check)
- Frontend has duplicate permission files with different values

## Scope

Fix **admin + critical paths only** (admin, booking, payment, review subgraphs).

---

## Phase 0: Unified Role Enum

Create `src/core/shared/domain/role.ts`:

```typescript
export enum Role {
  GUEST = "GUEST",
  CUSTOMER = "CUSTOMER",
  STAFF = "STAFF",
  MODERATOR = "MODERATOR",
  AGENT = "AGENT",
  OWNER = "OWNER",
  ADMIN = "ADMIN",
  SUPER_ADMIN = "SUPER_ADMIN",
}

export const ROLE_HIERARCHY: Record<Role, number> = {
  GUEST: 0, CUSTOMER: 1, STAFF: 2, MODERATOR: 3,
  AGENT: 4, OWNER: 5, ADMIN: 6, SUPER_ADMIN: 7,
};

export function hasMinRole(userRole: Role, requiredRole: Role): boolean {
  return (ROLE_HIERARCHY[userRole] ?? 0) >= (ROLE_HIERARCHY[requiredRole]);
}

export function mapLegacyRole(legacy: string): Role {
  const map: Record<string, Role> = {
    SUPER_ADMIN: Role.SUPER_ADMIN, ADMIN: Role.ADMIN, STAFF: Role.STAFF,
    AGENT: Role.AGENT, CUSTOMER: Role.CUSTOMER, USER: Role.CUSTOMER,
    OWNER: Role.OWNER, MEMBER: Role.CUSTOMER, TENANT_ADMIN: Role.ADMIN,
    MODERATOR: Role.MODERATOR,
  };
  return map[legacy] ?? Role.CUSTOMER;
}
```

Then update old enum files to re-export from here (alias pattern):
- `src/core/admin/domain/entities/adminRole.ts`
- `src/core/user/domain/userRole.ts`
- `src/core/tenant/infrastructure/models/member.ship.role.ts`

## Phase 1: Gateway Auth Enforcement

### 1a. Operation Auth Map — `src/gateway/auth/operationMap.ts` (NEW)

Code-based registry mapping operation names to auth requirements. Unlisted operations remain public (backwards-compatible).

```typescript
export interface OperationAuthRequirement {
  authenticated: boolean;
  minRole?: Role;
  permissions?: string[];
}
```

Key entries:
| Operation | authenticated | minRole |
|-----------|:---:|---|
| `adminUsers`, `adminUser`, `dashboardStats`, `systemSettings` | yes | ADMIN |
| `createAdminUser`, `updateAdminUser` | yes | ADMIN |
| `deleteAdminUser`, `deleteSystemSetting` | yes | SUPER_ADMIN |
| `myBookings`, `createBooking`, `cancelBooking` | yes | CUSTOMER |
| `confirmBooking`, `checkInBooking`, `completeBooking` | yes | OWNER |
| `analyzeBookingFraud` | yes | ADMIN |
| Payment mutations | yes | CUSTOMER |
| Review write mutations | yes | CUSTOMER |
| `oauthLogin`, `refreshToken` | no (public) | — |

### 1b. Auth Plugin — `src/gateway/plugins/auth.plugin.ts` (NEW)

Apollo Server plugin intercepting `requestDidStart`:
1. Decode JWT from `Authorization` header (reuse `getUserFromContext`)
2. Look up operation in auth map
3. Reject with `UNAUTHENTICATED` if no user + operation requires auth
4. Reject with `FORBIDDEN` if user role < required role
5. Attach decoded user to gateway context

### 1c. Gateway Context — `src/gateway/index.ts` (MODIFY)

- Register auth plugin
- Forward decoded user to subgraphs via `x-gateway-user` header in `willSendRequest`

### 1d. getUserFromContext — `src/infrastructure/auth/getUserFromContext.ts` (MODIFY)

Check `x-gateway-user` header first (pre-decoded by gateway), fall back to JWT decode.

## Phase 2: Policy Engine Wiring

### 2a. Fix `src/rbac/policy.engine.ts` (MODIFY)

- Fix broken imports (`@/domain/user/types/role` does not exist → use `@/core/shared/domain/role`)
- Extend rules for BOOKING, PAYMENT, REVIEW resources
- Use unified `Role` enum

### 2b. Create `src/infrastructure/auth/withAuthorization.ts` (NEW)

```typescript
export function withAuthorization<TArgs>(
  action: Action, resource: Resource,
  resolverFn: Resolver<TArgs>,
  options?: { resolveOwnerId?: (ctx, args) => Promise<string | null> }
)
```

Wraps a resolver: checks `context.user` exists, runs `PolicyEngine.can()`, throws `FORBIDDEN` on deny.

### 2c. Update `src/infrastructure/auth/require.auth.ts` (MODIFY)

Use `GraphQLError` with `UNAUTHENTICATED` extension code instead of plain `Error`.
